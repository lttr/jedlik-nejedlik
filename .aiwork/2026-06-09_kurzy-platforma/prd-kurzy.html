Cíle a hranice verze 1

✓ V rozsahu

  • Prodej digitálních kurzů: jednorázový nákup, trvalý přístup
  • Průběh: přihlášení → nákup → zpřístupnění kurzu
  • Postupné procházení. Odemykání sekcí přes testy (volitelná funkce per-sekce)
  • Bezpečné zpřístupnění videí (nejdou snadno sdílet dál)
  • Napojení na fakturační systém s automatickým vystavením faktury jménem spolku po platbě
  • Platby v CZK, čeština, malá rostoucí nabídka kurzů

✗ Mimo rozsah

  • Fyzické produkty, doprava, sklad
  • Firemní nebo hromadné licence (více míst)
  • Předplatné nebo opakované platby
  • Vyhledávání, doporučování, marketingová automatizace
  • Více měn nebo ceny mimo CZK · jiný jazyk než čeština
  • Certifikáty (později) · LLM hodnocení otevřených odpovědí (příští verze)

Učiněná rozhodnutí

Body rozhodnuté z analýzy a grilování (byznysové).

R-1

Vzdělávací platforma, ne e-shop. Přidaná hodnota = správa přístupu, postup, bezpečná videa.

R-2

Verze 1 jen digitální. Datový model připraven na pozdější fyzické, ale teď se nestaví.

R-3

Jednorázový nákup, trvalý přístup. Žádné předplatné (zbytečně komplikuje).

R-4

Zachovat platební bránu, ukončit formulářový systém. Brána po platbě notifikuje server → spolehlivé „zpřístupni kurz".

R-5

Přístup vynucen na serveru. Skrytí ve frontendu není ochrana. Evidence „uživatel × kurz × postup".

R-6

Úspěšný test odemyká trvale. Splnění se uloží. Při návratu se sekce znovu nezamyká.

R-7

Prodejcem je spolek. Na každé faktuře IČO a název spolku, bez ohledu na to, kdo videa natočil.

R-8

Zatím neplátce DPH, ale s ohledem na hranici. Příjem z kurzů se do hranice počítá (na rozdíl od darů).

Mechanika kurzů jádro projektu

Obsah je po nákupu z velké části přístupný hned. Vybrané sekce se odemykají podle pravidla na úrovni sekce.

Tři pravidla odemčení sekce

a) Test

Jediná automatická brána. Sekce může obsahovat test, ten může být blokující (pole blocking). Blokující je podmínka postupu, jinak jde o nezávazné procvičení.

b) Ručně adminem

Ruční odemčení administrátorem. Sem spadá i ručně hodnocená práce (žádná samostatná entita „úkol").

c) Čas od nákupu

Po uplynutí času od udělení oprávnění. Hodiny každého studenta běží od jeho koupě (každý student zvlášť, ne absolutní datum).

Testy

  • Hranice úspěšnosti nastavitelná pro každý kurz
  • Otázky typu výběr z možností (deterministicky vyhodnotitelné)
  • Neomezeně pokusů, bez prodlevy
  • Otevřené odpovědi + LLM hodnocení → příští verze

Identita & přístup

  • Account-first: registrace nebo přihlášení před nákupem
  • Identitou je e-mail
  • Objednávka i oprávnění se váží na známého studenta
  • Magic link odložen (Directus to nativně neumí)

Platby & fakturace

Serverová notifikace je zdroj pravdy, ne návrat prohlížeče. Vše idempotentní.

Objednávka → redirect → GoPay (CZK) ⤳ notifikace ⤳ Server (Nitro)

Po potvrzené platbě server idempotentně (dle ID platby):

1. Udělí oprávnění
zpřístupní kurz studentovi
2. Vystaví fakturu
Fakturoid API v3 (~3 requesty)
3. Pošle potvrzení
Fakturoid odešle fakturu = potvrzení

Opuštěné nebo neúspěšné platby zůstanou jako nezaplacená objednávka bez oprávnění a faktury. Návratová URL slouží jen k UX.

GoPay platební brána

Nativně v CZK, účet pod spolkem připraven. Redirect + serverová notifikace. Idempotence dle ID platby (notifikace se opakují).

Fakturoid fakturace

API v3, OAuth 2, tarif Na lehko. Jediný zdroj číselné řady. Jedna platba = jedna faktura (ID faktury u objednávky). Hlídá i DPH hranici.

DPH: sledování hranice

Zatím neplátce. Faktury bez DPH s označením „neplátce DPH". Sledování řeší Fakturoid (žádné vlastní počítadlo).

Architektura Část B

Záměrně jednoduchá. Directus drží data i hranici. Nitro je tenká důvěryhodná vrstva.

Directus

systém záznamu + hranice

Obsah i transakční data (studenti, objednávky, oprávnění, postup, pokusy). Poskytovatel identity i primární brána přístupu (oprávnění Directu). Dělá on-the-fly transformace obrázků.

Nitro

tenká důvěryhodná vrstva

Jen operace s tajným klíčem nebo serverovou logikou: podpis URL videa, GoPay webhook, volání Fakturoidu, příjem podkladů. Data vždy ukládá zpět do Directu. Nikdy druhý systém záznamu.

Jeden nasazovaný celek z Nuxt vrstev TR-1b

Žádná druhá aplikace. Stávající web/ (Node/Nitro SSR, Nixpacks → Coolify) se člení na vrstvy, jedna na oblast. Společný ohraničený kontext, sdílený jazyk.

customers
identita / autentizace
lms
kurzy, sekce, lekce, postup, testy, video
shop
katalog, checkout, platby, fakturace

Technologický stack

Nuxt + Vue 3 + TS Directus (CMS) Nitro GoPay (CZK) Fakturoid API v3 (OAuth2) Cloudflare Stream Cloudflare + VPS monorepo
Video (TR-2): nenahrává se do Directu. Lekce drží jen ID videa ve Stream. Nitro po ověření oprávnění vydá krátkodobý podepsaný token k přehrání.

Nahrávání podkladů součást v1.0

Jedna serverová logika v Nitro, dvě tenká rozhraní. Routuje podle MIME typu.

obrázky / PDF / dokumenty → Directus files (volitelně sharp preprocess)
video → Cloudflare Stream (Direct Creator Upload + ready webhook)

Prohlížeč/CLI nahrává video přímo do Streamu (tus URL), ne přes náš server. Ověření: volající je Directus uživatel s rolí autor/admin.

a) Web admin stránka

Přihlášení jako Directus uživatel, drag-and-drop, průběh + stav „ready".

b) CLI Deno + dax

Jeden příkaz upload <cesta> [--lesson X] — pro Claude Code. MCP zvažováno a odloženo, endpointy ale MCP-ready (TR-7b).

Zvažované a zamítnuté alternativy

Proč ne jiné nástroje.

zamítnuto

Fakturovac.cz (stávající)

Nemá API, jen ruční PDF. Nelze automatizovat vystavení faktury po platbě.

zamítnuto (těsně)

Vyfakturuj.cz

Reálná alternativa k Fakturoidu (API v2, sleva pro neziskovky, příbuznost se SimpleShop). Fakturoid vyhrál kvůli lepší dokumentaci a API ve všech tarifech. Rozdíl malý — lze se vrátit.

zamítnuto

FAPI (celková platforma)

LMS část je WordPress plugin (mimo stack) a neumí test-gated progresi (odemyká podle platby). Unikátní jádro (LMS s testy) by stejně bylo nutné psát vlastní.

odloženo

Stripe místo GoPay

Lepší EU OSS/DPH automatizace a předplatné. GoPay je zavedená, nativně v CZK a pro jednorázový nákup stačí. Relevantní až při expanzi do EU.
